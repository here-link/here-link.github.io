◆ Dev_BleChangeName()

int Dev_BleChangeName ( uint32_t  Timeout)

Timeout(秒)时间之后更改蓝牙名称

Returns
0:成功；1:失败
Note
需要调用HL_GetBleName获取蓝牙名称。这个函数不能阻塞，以便让蓝牙数据发送完成，建议启动一个定时器。 定时器到期后如果蓝牙正在广播就重新发起广播，如果蓝牙还在连接态就等断开；每次开始广播时要重新获取蓝牙名称。

◆ Dev_BleDisable()

void Dev_BleDisable ( void  )

禁止蓝牙功能，不进行广播，不接受连接。不会在蓝牙连接状态调用，会在发送网络数据之前调用

这个目的是为了避免蓝牙通道数据和其他网络通道的数据产生冲突，但是也不能因为网络情况较差导致蓝牙长时间不可用，所以建议延迟处理， 具体的是在网络模块连接网络成功后开始发送数据前再执行这个动作

Cpu card 发送数据

Parameters

Returns

Example

int Dev_CpuCardApdu(uint8_t* command, uint16_t commandLen, uint8_t* response, uint8_t* responseLen, uint16_t *apdusw)
{
uint8_t receOffset = 1; //接收数据中协议字节长度
uint8_t sendBuf[48] = {0}, receBuf[32] = {0}, receLen = 0;
uint16_t sendLen = 0;
sendBuf[0] = CPU_CARD.PCB;
memcpy(sendBuf + 1, command, (uint32_t)commandLen);
sendLen = 1 + commandLen;
//调用门锁厂商APDU指令接口
if(FM175XX_SUCCESS != CPU_TPDU(sendLen, sendBuf, &receLen, receBuf))
{
return 1;
}
else
{
*apdusw |= (uint16_t)(*(receBuf + receLen - 2)) << 8;
*apdusw |= (uint16_t)(*(receBuf + receLen - 2 + 1));
CPU_CARD.PCB ^= 1; //PCB = 02/03
memcpy(response, receBuf + receOffset, receLen - receOffset - 2); //2表示返回码长度
*responseLen = receLen - receOffset - 2;
return 0;
}
}
